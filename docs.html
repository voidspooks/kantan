Kantan Docs

How to use Kantan, a small text editor for macOS.

Introduction

Kantan is a native macOS text editor written in Swift. It opens files, edits them, saves them. It highlights syntax, shows line numbers, marks git changes in the gutter, and lets you split the window in two. That's roughly the whole feature set, and it's intentional.

The philosophy: editors grow heavy as features pile up. Kantan resists this. If a feature does not earn its place, it does not belong.

If you're looking for plugins, AI integration, language servers, or a settings UI with hundreds of toggles, Kantan is not for you. If you want a fast, dark, keyboard-driven editor that opens instantly and stays out of your way, read on.

Install & build

Kantan ships as source. You build it locally with the system Swift toolchain — no package manager, no Xcode project, just swiftc and a Makefile.

Requirements

  • macOS with the Xcode Command Line Tools installed (xcode-select --install).
  • The swiftc compiler on your PATH.

Build

From the project root:

make

This compiles every .swift file under src/ into Kantan.app/Contents/MacOS/Kantan, copies Info.plist into the bundle, and generates AppIcon.icns from icon.png.

Run

make run

This builds (if needed) and opens Kantan.app. You can also drag the .app bundle into /Applications and launch it from there.

Clean

make clean

Removes the built app, the build/ directory, and the generated icon files.

First run

The first time Kantan launches, it creates a config file at:

~/Library/Application Support/Kantan/settings.yaml

The file is seeded with sensible defaults — line numbers on, dark background, per-language indent rules, and a syntax highlighting palette for every supported language. You don't need to touch it to get started; it's there if you want to customize anything later. See settings.yaml for details.

An empty Untitled tab opens automatically. Start typing, or press ⌘O to open a file.

Opening files & folders

Kantan opens individual files and entire folders. A folder open populates the left sidebar with a file tree; an individual file open just adds a tab.

Open a file

⌘O, or File → Open…. Pick a file in the panel.

If the file is already open in any pane, Kantan focuses its existing tab instead of opening a duplicate.

Open a folder

⌘⇧O, or File → Open Folder…. Pick a directory.

The folder appears in the sidebar. Click any file to open it as a tab. The sidebar persists across launches — see session restore.

Save

  • ⌘S — save the current tab. If the tab is Untitled, this opens a Save As panel.
  • ⌘⇧S — save the current tab to a new path.

Saving an Untitled tab to a known extension automatically enables syntax highlighting for that language.

New & close

  • ⌘N — new Untitled tab.
  • ⌘W — close the active tab. If the tab has unsaved changes, you'll be prompted to save, discard, or cancel.
  • ⌘⇧W — close the entire window. Same prompt for any dirty tab.

Tabs

Each pane has its own tab bar across the top. A dot to the left of a tab title means the tab has unsaved changes.

Switch tabs

  • Click a tab to focus it.
  • ⌘⇧] — next tab.
  • ⌘⇧[ — previous tab.

Reorder tabs

Drag a tab left or right within the tab bar to reorder it.

Tab context menu

Right-click any tab to split the window with that tab — see split panes.

Split panes

Kantan supports a single split — two panes side-by-side or stacked. Each pane has its own tab list, scroll position, and active tab.

Create a split

  • Right-click a tab and choose Split Vertically or Split Horizontally. The chosen tab moves into the new pane.

Move tabs between panes

Drag a tab off its tab bar and drop it onto the other pane's tab bar. The tab is transferred (with its undo history, scroll position, and dirty state) to the destination pane at the drop position.

Collapse the split

Close every tab in one pane. When a pane empties, it disappears and the remaining pane fills the window.

Editing

Editing is plain NSTextView behavior with a few additions.

Auto-pair

Typing {, [, (, `, ", or ' auto-inserts the closing character and places the cursor between them. Quotes are not auto-paired when the previous character is a letter, digit, or underscore — so you can type don't without fighting the editor.

If you have a selection when you press an opener, the selection is wrapped in the pair instead.

Auto-indent

Pressing Return carries the leading whitespace of the current line onto the new line. If the line ends in {, [, or (, an extra indent is added — using the per-language indent style and width from settings.yaml. See indent style.

Undo & redo

Each tab has its own undo manager, so undo on one tab never replays edits from another.

  • ⌘Z — undo.
  • ⌘⇧Z — redo.

Cut, copy, paste, select all

Standard system shortcuts: ⌘X, ⌘C, ⌘V, ⌘A.

Find & replace

⌘F opens the find bar at the top of the current pane. The bar uses the system's incremental find — matches are highlighted as you type.

The bar's mode toggle switches between Find and Find & Replace. The standard system controls (next, previous, replace, replace all, regex, case-sensitive, wrap) are all available.

Sidebar & file tree

When you open a folder, its contents appear in the sidebar on the left. Hidden files (dotfiles) are not shown.

Toggle the sidebar

⌘B, or Configuration → Toggle Sidebar.

Open files

Click any file row to open it in the focused pane.

Expand & collapse folders

Click a folder row to toggle expansion. Sub-trees stay loaded across refreshes.

Right-click menu

Right-click a row (or empty sidebar space) for these actions:

  • Copy — copy the file/folder URL to the system pasteboard.
  • Paste — paste files from the pasteboard into the clicked directory. Name collisions get a file 2.txt-style suffix.
  • Rename — inline-edit the name. Press Return to commit, Esc to cancel.
  • New File / New Folder — only on directories (and empty space). Creates a placeholder and drops you into rename mode.
  • Delete — confirms, then moves to Trash.

External changes

The sidebar watches the project directory with FSEvents. When files change on disk (other editors, git checkout, build tools), the tree and git status refresh automatically after a brief debounce. Press Configuration → Refresh Sidebar to force an immediate refresh.

Footer

The bottom of the sidebar shows the current cursor position (Ln 12, Col 4) and, if the project root is a git repo, the current branch.

Git integration

Kantan is git-aware in two small ways. There are no commands to run, no buttons to push — git state is just visible at a glance.

Sidebar status colors

When the project root is a git repository, file rows are colored according to git status --porcelain:

  • Untracked files are tinted (and untracked directories cascade their state to every file inside).
  • Modified files are tinted differently.

The status snapshot refreshes when the file tree changes, when you save a file, or when FSEvents reports a directory change.

Branch indicator

The sidebar footer shows the current branch name, refreshed alongside the status snapshot. The footer hides itself when the project root isn't a git repo.

Gutter diff strips

To the left of the line numbers, Kantan draws colored strips for lines that differ from HEAD. The strips recompute on file open and on save, so they stay accurate as you edit.

Syntax highlighting

Highlighting is built in for 25 languages — see supported languages. The highlighters are simple regex-driven passes; they're fast and unobtrusive, not semantically perfect.

Auto-detection

When you open a file, Kantan picks a language from the file extension (or, for Makefile, the filename). If a language is detected and highlighting is currently off, it's switched on automatically.

Toggle highlighting

Configuration → Syntax Highlighting.

Override the language

Configuration → Language shows every supported language with a checkmark next to the active one. Pick another to re-highlight the active tab against that grammar — useful for files with non-standard extensions.

Recolor on settings change

If you save changes to settings.yaml from inside Kantan, the syntax palette reloads and every open tab is re-highlighted in place. No restart.

Font size

Kantan uses Menlo by default (falling back to the system monospace font). The size lives between 8pt and 32pt.

  • ⌘+ — increase text size.
  • ⌘- — decrease text size.

The size applies to every pane, the line-number gutter, the tab bar, and the sidebar at once.

Session restore

Kantan remembers the last folder and last file you had open, in UserDefaults. On launch, it restores both — the folder repopulates the sidebar, and the file opens into the first tab. If neither is set (or either has been deleted), it opens a fresh Untitled tab.

settings.yaml

Configuration lives in a single YAML file:

~/Library/Application Support/Kantan/settings.yaml

Open it from Kantan with Configuration → Settings. Saving the file from inside Kantan reloads the configuration and applies it to every open tab without a restart.

The file has four top-level sections: line_numbers, theme, indent, and syntax_highlighting. The parser is forgiving — anything outside these shapes is silently ignored, and missing keys fall back to defaults.

Theme colors

The theme block currently controls two backgrounds:

theme:
  editor_background:  "#000000"
  sidebar_background: "#000000"

Values are "#RRGGBB" hex strings. Anything else is ignored. Foreground text colors live in the syntax_highlighting block.

Indent style

The indent block sets per-language indent character (tab or space) and width:

indent:
  swift:
    style: space
    width: 4
  go:
    style: tab
    width: 1
  yaml:
    style: space
    width: 2

The values are used by the auto-indent rule that triggers after a line ending in {, [, or (. If a language is missing or malformed, Kantan falls back to four spaces.

Syntax colors

Each supported language has its own block under syntax_highlighting, mapping token kinds to hex colors:

syntax_highlighting:
  ruby:
    keyword:  "#c685c0"
    string:   "#ce9178"
    comment:  "#6b9955"
    number:   "#b5cea8"
    constant: "#4ec9b0"
    symbol:   "#569cd6"
    variable: "#9cdcfe"

Token names vary slightly by language — Markdown has heading, strong, blockquote; HTML has tag, attribute; CSS has selector, property; and so on. The defaults seeded into settings.yaml on first launch are the canonical reference for what's available.

Unknown token names and malformed colors are ignored, so you can edit one value at a time without breaking the rest.

Line numbers

The top-level line_numbers key toggles the gutter:

line_numbers: true

You can also flip it from the menu: Configuration → Show Line Numbers (⌘⇧L). Toggling from the menu rewrites settings.yaml in place — preserving any trailing comment on that line — so the change persists across launches.

Troubleshooting

I edited settings.yaml but nothing changed

Settings reload when the file is saved from inside Kantan. If you edited it with another tool, open Configuration → Settings and save it (⌘S) once — Kantan will pick up the new values. Or quit and relaunch.

Syntax highlighting looks wrong for a file

Force the language with Configuration → Language. Kantan picks a language from the extension; if your file has a non-standard extension, the auto-detect won't fire.

The git footer is missing

The branch indicator only shows when the project root is a git repository. If you opened a sub-directory of a repo as the root, Kantan won't recognize it as a repo — open the repo root instead.

The sidebar didn't update after I created a file outside Kantan

Kantan watches with FSEvents and refreshes after a one-second debounce. If it still hasn't picked up a change, run Configuration → Refresh Sidebar.

Build fails with "no such file or directory: swiftc"

Install the Xcode Command Line Tools: xcode-select --install.
